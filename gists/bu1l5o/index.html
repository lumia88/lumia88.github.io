less-loader引入普通css文件

在 xxx.less 文件中引入 xxx.css 的正确姿势

然而，在执行依赖包安装完毕之后，运行项目的开发，出现了这样的错误：

报错原因：在 index.less 中直接引入了.css文件，当 less-loader 以 less 的编译规则对 css 文件进行编译后，重命名文件前后的名称、后缀名都没有变化，导致 less-loader 无法判断编译是否完成，从而报错。

解决方法：在.less 文件中引入.css 文件时，添加编译设置让 less-loader 直接引入文件并不对文件进行任何编译。如：@import (inline) './base.css';

reference: 使用该文件，但不编译输出到目录；
inline: 仅引入对应的样式文件，不对样式文件做任何编译，保留原始内容；
less: 忽略文件的类型，强制采用 less-loader 的规则进行编译；
css: 忽略文件的类型，强制采用 css 文件的方式进行编译和引入；
once: 只引入一次该文件（默认配置）；
multiple: 对此引入该文件？【不清楚多次引入的目的——】；
optional: 指明该文件为可选文件，当无法搜寻到文件时跳过该文件的编译步骤。
